## 2. Аналитическое решение задачи регрессии

### 2.1 Векторная форма уравнения

Задача линейной регрессии формулируется как поиск вектора весов **w**, минимизирующего функцию потерь:

**Матричная форма нормального уравнения:**

```
w = (X^T X)^{-1} X^T y
```

где:

- **X** — матрица признаков размерности (n × m)
- **y** — вектор целевых значений размерности (n × 1)
- **w** — вектор весов размерности (m × 1)

### 2.2 Изменения при добавлении регуляризации

#### L2-регуляризация (Ridge)

Функция потерь:

```
L(w) = ||y - Xw||² + α ||w||²
```

Аналитическое решение:

```
w = (X^T X + αI)^{-1} X^T y
```

**Эффект:** веса сжимаются к нулю, но не становятся точно нулевыми.

#### L1-регуляризация (Lasso)

Функция потерь:

```
L(w) = ||y - Xw||² + α ||w||₁
```

**Особенность:** не имеет аналитического решения в замкнутой форме, решается методами оптимизации (например, coordinate descent).

### 2.3 Почему L1-регуляризация используется для отбора признаков

L1-регуляризация создает **разреженные веса** (sparse weights) благодаря особенности L1-нормы:

1. **Геометрическая интерпретация:** L1-норма имеет "углы" на осях координат, что приводит к обнулению некоторых весов
2. **Математическая причина:** субградиент L1-нормы содержит ноль в точке w=0, что позволяет оптимизационному алгоритму "застрять" на нулевых значениях
3. **Практический эффект:** признаки с нулевыми весами можно исключить из модели, упрощая интерпретацию и снижая риск переобучения

### 2.4 Учет нелинейных зависимостей

Одни и те же линейные модели (Linear Regression, Ridge, Lasso) могут моделировать нелинейные зависимости через:

1. **Полиномиальные признаки:** добавление степеней и взаимодействий исходных признаков

   ```python
   poly = PolynomialFeatures(degree=d)
   X_poly = poly.fit_transform(X)
   ```
2. **Базисные функции:** использование функций-преобразований (sin, cos, exp, log)
3. **Ядерные методы:** Kernel Ridge Regression (KRR)

## 4. Реализация моделей

### 4.1 Линейная регрессия (SGD)

```python
class MyLinearRegressionGD:
    def __init__(self, learning_rate=0.01, n_iter=1000, random_state=21):
        self.lr = learning_rate
        self.n_iter = n_iter
        np.random.seed(random_state)  # Детерминированность
      
    def fit(self, X, y):
        self.weights = np.random.randn(X.shape[1])
        for _ in range(self.n_iter):
            grad = -2 * X.T.dot(y - X.dot(self.weights)) / len(y)
            self.weights -= self.lr * grad
        return self
```

**Детерминированность модели:** модель является детерминированной, если при одинаковых начальных условиях и одних и тех же входных данных она всегда выдает одинаковые результаты. В SGD детерминированность достигается:

- Фиксацией random_state
- Отсутствием случайного перемешивания данных (или фиксацией перемешивания)

### 4.2 Метрики качества

```python
def r2_score(y_true, y_pred):
    ss_res = np.sum((y_true - y_pred) ** 2)
    ss_tot = np.sum((y_true - np.mean(y_true)) ** 2)
    return 1 - (ss_res / ss_tot)
```

## 5. Нормализация признаков

### 5.1 Зачем нужна нормализация?

**Обязательна:**

- Для методов, чувствительных к масштабу (градиентный спуск, SVM, нейронные сети)
- Для методов, использующих расстояния (kNN, k-means)
- При использовании регуляризации (L1, L2) — веса признаков в разных масштабах штрафуются неравномерно

**Необязательна:**

- Для деревьев решений и ансамблевых методов
- Для линейной регрессии без регуляризации

### 5.2 MinMaxScaler

**Формула:**

```
X_scaled = (X - X_min) / (X_max - X_min)
```

**Результат:** значения в диапазоне [0, 1]

### 5.3 StandardScaler

**Формула:**

```
X_scaled = (X - μ) / σ
```

**Результат:** среднее = 0, стандартное отклонение = 1

### 10.2 Удаление выбросов

**Почему выбросы удаляются только из обучающей выборки:**

- Тестовая выборка должна отражать реальное распределение данных
- Удаление выбросов из теста даст нечестную оценку качества
- Модель должна быть устойчива к выбросам, которые могут встретиться в реальных данных
